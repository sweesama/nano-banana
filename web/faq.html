What is Nano Banana?
Nano Banana is an AI image editing brain powered by Gemini 3.0. Unlike traditional "filters," it understands natural language instructions (e.g., "Make the cat wear sunglasses") to perform precise, context-aware edits.
Can I run Nano Banana locally?
Yes and No. The "Pro" model (Gemini 3.0) is closed-source and cannot run totally offline. However, you can run the Nano Banana Client locally via Python to control it from your terminal.

If you need a 100% offline solution (e.g., for air-gapped security), check out our Hardware Guide for alternative models like Z-Image Turbo (Requires 16GB+ VRAM).
Is it free?
The local Python Client uses Google's Gemini API, which currently offers a generous free tier for experimentation. Heavy commercial use may require a paid billing account with Google Cloud.
How fast is it?
The Pro Client takes about 5-10 seconds per edit (Cloud Latency).
Offline Models (Z-Image) can generate in ~2 seconds on an RTX 4090, or ~2 minutes on a CPU.
How do I keep character consistency?
Consistency is Nano Banana's main strength over models like Flux. Our "Instruction Mode" preserves structural identity by default. For advanced consistency, check our Recipes for "Outfit Change" workflows.
Where is the API Key?
You can get your key from Google AI Studio. Once you have it, set `export GOOGLE_API_KEY=your_key` in your terminal to empower the client.
Can I use images commercially?
Images generated via Gemini 3.0 (Pro) are generally yours to own, but subject to Google's Generative AI terms. Offline models (Z-Image) are often Apache 2.0. Always verify the specific model license before commercial deployment.
How do I reproduce benchmark results?
Each benchmark page lists the exact Prompt, Seed, and Model Version. If using the Cloud Client, results may vary slightly over time as the base model updates. Offline models offer bit-exact reproducibility.
