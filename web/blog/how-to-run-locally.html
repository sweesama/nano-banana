The "Closed Source" Reality

Before we begin, a crucial clarification: Nano Banana Pro cannot truly run offline.

Why? Because the core model is **closed source**. It runs on Google's Gemini 3.0 infrastructure. When you "run it locally," you are actually running a local client that sends instructions to our cloud brain.

Step 1: Get Your API Key

Since the logic happens in the cloud, you need a key to unlock it.

  1. Visit Google AI Studio.
  2. Click "Get API Key".
  3. Copy it to a safe place.

Step 2: Prepare Your Environment

You don't need a GPU, but you do need Python.

# Check if you have Python installed
python --version

# Create a clean workspace
mkdir nano-client
cd nano-client
python -m venv venv
# On Windows:
.\venv\Scripts\activate
# On Mac/Linux:
source venv/bin/activate

Step 3: Install the Client SDK

We use the official Google GenAI SDK to communicate with the Pro model.

pip install google-genai pillow

Step 4: The Core Script

Save the following as nano_api.py:

import os
from google import genai
from PIL import Image

# Paste your key here (or better, use environment variables)
os.environ["GOOGLE_API_KEY"] = "YOUR_KEY_HERE"

client = genai.Client(api_key=os.environ["GOOGLE_API_KEY"])

print("Connecting to Nano Banana Pro (Cloud)...")

Want True Offline?

If you strictly need to run without internet (e.g., for air-gapped security), you cannot use Nano Banana Pro. Instead, check out the Z-Image Turbo alternative in our Quickstart Guide. It is open-source and runs completely offline, provided you have an RTX 4080 or better.
